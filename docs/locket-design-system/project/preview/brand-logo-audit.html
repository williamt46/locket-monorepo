Issues

Blocker — Transparent mark is not transparent

locket-mark-transparent.svg has fill="#FDFBF9" on the inner circle — identical to the light mark. On blue or dark backgrounds (see row 1, columns 4–5 above) the sand inner circle is clearly visible. For true transparency the inner fill should be "none". Currently this file is a duplicate of the light mark, not a usable transparent asset.

Blocker — Lockup wordmark has no embedded font

Both lockup SVGs have an empty <defs> block and use <text class="wm">locket</text> with no font-family, font-size, or font-weight — and no .wm style defined anywhere. The text will render in whatever the host system's default serif/sans is. To fix: either embed a @font-face for Public Sans in <defs><style>, or convert the wordmark to outlines (preferred for a logo master file).

Undocumented color — #1A2332 navy

The dark mark inner well and the lockup-dark background use #1A2332 — a deep navy that doesn't exist in the design system token set (dark bg is #252628, dark card is #323336). It reads beautifully as the mark's inner fill against the charcoal dark background, but it needs a token: suggest --locket-navy. Alternatively, reconcile with #252628 for consistency.

Lockup alignment inconsistency

Light lockup: mark at translate(0,0), text at x="60". Dark lockup: mark at translate(8,0), text at x="68". Both preserve the 60px gap from mark origin to text, so the visual spacing is equivalent — but the dark background rect starts at 0 while the mark has 8px left padding. This is intentional (dark badge needs breathing room) but not documented. Worth noting if these are used at small sizes.

Good — Mono-white is actually a badge, not pure mono

locket-mark-mono-white.svg has fill="#1A2332" on the inner circle — all strokes and icons are white, but the well is navy, creating a strong dark inset. This is visually effective (the mark reads as a pin/badge) but "mono-white" is slightly misleading. Consider renaming to locket-mark-badge-dark to clarify intent.

Missing variants

Set doesn't include: lockup-mono-white.svg, lockup-mono-ink.svg, or an app icon master (mark on a background for App Store / Play Store). These are needed before the brand kit is production-complete.
