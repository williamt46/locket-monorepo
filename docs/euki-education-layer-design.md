## Problem Statement

Locket has zero education content. The calendar and encrypted log exist, but users who don't already understand their cycle phases, symptoms, or product options get no guidance. Euki — a nonprofit, GPL v3.0 open source iOS app backed by Consumer Reports and privacy analysts — has built a high-quality, offline-first, evidence-based reproductive health content library. Locket can integrate the cycle-core subset (menstruation 101, product guide, symptom management) contextually — no dedicated tab, content surfaces inline where users are already engaged.

---

## What Makes This Cool

Locket already computes cycle phase via `usePredictions`. That phase is live data. Pairing it with Euki's content makes the calendar feel alive — not just a log, but a guide. A user in the luteal phase sees bloating and mood content before they need to search for it. A user who just logged cramps gets a "why does this happen?" link in the same tap. That's the unlock: the app knows where you are and tells you something useful about it.

It's also a strong privacy-aligned move. Euki's content is entirely offline — no analytics, no CDN calls. Bundled JSON works the same way Locket's crypto layer works: everything local, nothing phoned home.

## Constraints

- No new navigation tab in MVP — content surfaces inline only
- All content must be bundled locally (offline-first, privacy-first)
- Design system: Sun-Baked Sand bg (#FDFBF9), Locket Blue (#006EC7), cycle phase colors from `docs/design.md`, Material Symbols icons, SF Pro / system fonts
- React Native / TypeScript stack; Euki is Swift — port content data only, no Swift code
- Must not break existing DataEntryModal behavior or LedgerScreen layout
- Medical disclaimer required before first content view (one-time acknowledgment)

### Legal Note on Euki Content (GPL v3.0)

Euki's JSON content files and Localizable.strings are text/data, not compiled software. However, if `menstruation.ts` is compiled into the app binary, the FSF's "part of the program" standard may apply. The accurate position:

1. Add a `NOTICE` file crediting Euki, Inc. and linking to the source repo.
2. For an open-source Locket: GPL v3.0 compliance is straightforward (attribution + source available).
3. For a future closed-source Locket: get a legal opinion before shipping.

Required in-app attribution: `Educational content from Euki (euki.app) — open source reproductive health education.`

**Content versioning:** Euki's content is a versioned snapshot. Locket pins to a specific Euki commit SHA at time of port. Review cadence: check `Euki-Inc/Euki-iOS` for content file changes every 6 months, or when Euki tags a new release. There is no live dependency or auto-sync — offline-first reliability is the deliberate tradeoff. Add to `NOTICE` file: `Euki content snapshot from commit [SHA], [date]. Review for updates at github.com/Euki-Inc/Euki-iOS.`

## Premises

1. **The DataEntryModal is the primary injection surface.** Users are actively thinking about their body at log time.
2. **Symptom fields are a prerequisite.** Content about cramps is useless if users can't log cramps. Bleeding intensity + symptom chips ship together with the content layer.
3. **Euki's content is portable, but requires a merge step.** The JSON is structural (IDs, section hierarchy). Actual text lives in `Localizable.strings`. Porting = merge both into a single TypeScript-friendly content module. Not just copying JSON.
4. **No new navigation = no cognitive overhead.** A Learn tab creates a decision split. Inline cards eliminate it.
5. **Cycle-phase awareness is the unlock — but requires a hand-authored phase mapping.** Euki organizes content by topic (`symptom_management`, `menstrual_cycle_101`), not by phase. The mapping from CyclePhase → content item IDs is Locket's to author. Not automagic.

## Recommended Approach: Two Surfaces, One Content Layer

### Architecture

```
packages/shared/src/euki/
  content/
    menstruation.ts           ← merged content (JSON structure + resolved strings, GPL attribution comment)
  types.ts                    ← EukiSection, EukiItem, EukiLink types
  index.ts                    ← getEukiContent() singleton
  phaseMapping.ts             ← CyclePhase → EukiItem[] (hand-authored)
  symptomMapping.ts           ← SymptomKey → EukiItem (direct lookup)

apps/mobile/src/
  components/
    PhaseInsightCard.tsx      ← LedgerScreen: phase-matched snippet card
    SymptomChips.tsx          ← DataEntryModal: bleeding + symptom selection
    ContentSheet.tsx          ← shared bottom sheet for full content detail
    DisclaimerModal.tsx       ← one-time medical disclaimer acknowledgment
  hooks/
    useEukiContent.ts         ← hook: (phase, dayInCycle) → { phaseSnippet, getSymptomContent }
```

### Data Model Changes

`inscribe(data: any)` already accepts arbitrary JSON and encrypts the whole payload. No migration concern — existing encrypted records are unaffected. New fields are additive.

```typescript
// Extend existing log entry payload (additive — backward compatible)
interface LogEntry {
  event: 'period_start' | 'period_end' | 'manual_entry';
  date: string;
  ts: number;
  // NEW — all optional, existing logs without these fields decode normally:
  bleeding?: {
    intensity: 'spotting' | 'light' | 'medium' | 'heavy'; // from Euki: bleeding_size_1–4
    clots?: 'small' | 'large';                             // from Euki: bleeding_clots_1–2
    // Products deferred to v2 (keeps DataEntryModal uncluttered — see Open Questions #3)
  };
  symptoms?: SymptomKey[];  // see canonical list below
  note?: string;
}

// Canonical symptom list (reconciled from Euki Localizable.strings body_1–body_N)
// MVP ships 6 minimum — the 6 marked with * below. The remaining 3 are included in the
// type so the data model doesn't need a migration when v2 adds them.
type SymptomKey =
  | 'cramps'        // *
  | 'bloating'      // *
  | 'nausea_fatigue'// *
  | 'mood_low'      // *
  | 'mood_anxious'  // *
  | 'mood_irritable'// *
  | 'acne'          // v2
  | 'headache'      // v2
  | 'back_pain';    // v2
```

### Euki Content Schema (Actual — from source inspection)

The JSON files are structural; text lives in `Localizable.strings`. Here is the resolved schema for `menstruation_options.json` → `menstruation.ts`:

```typescript
// packages/shared/src/euki/types.ts
interface EukiItem {
  id: string;          // e.g. "cramps", "bloating_and_other_gi_effects"
  title: string;       // resolved English text, e.g. "Cramps"
  content: string;     // resolved English text paragraph
  links?: EukiLink[];  // external resources (Planned Parenthood, WHO, etc.)
}

interface EukiSection {
  id: string;          // e.g. "symptom_management", "menstrual_cycle_101"
  title: string;
  content: string;     // section intro paragraph
  items: EukiItem[];   // expandable sub-items
}

// packages/shared/src/euki/index.ts
interface EukiContent {
  sections: EukiSection[];
  getSection(id: string): EukiSection | undefined;
  getItem(id: string): EukiItem | undefined;  // searches across all sections
}

export function getEukiContent(): EukiContent { /* lazy singleton */ }
```

### Phase → Content Mapping (Hand-Authored)

```typescript
// packages/shared/src/euki/phaseMapping.ts
// CyclePhase values match usePredictions return type
// All IDs below are verified against menstruation_options.json expandable_items.
export const phaseMapping: Record<CyclePhase, string[]> = {
  menstrual:   ['cramps', 'bleeding', 'nausea_and_fatigue'],
  follicular:  ['menstruation_period', 'ovulation', 'know_your_body'],
  ovulatory:   ['ovulation', 'menstruation_period'],
  luteal:      ['bloating_and_other_gi_effects', 'mental_and_emotional_health', 'other_pain'],
  unknown:     ['menstruation_period'],  // fallback: most broadly useful item
  // Note: 'menstrual_products' is a section, not an item — link to it from ContentSheet footer
};

// packages/shared/src/euki/symptomMapping.ts
// All IDs are expandable_items IDs from symptom_management section in Euki source.
// IMPORTANT: 'menstrual_cycle_101' is a SECTION ID — never pass it to getItem().
// v2 symptoms (acne, headache, back_pain) map to the nearest valid item.
export const symptomMapping: Record<SymptomKey, string> = {
  cramps:         'cramps',
  bloating:       'bloating_and_other_gi_effects',
  nausea_fatigue: 'nausea_and_fatigue',
  mood_low:       'mental_and_emotional_health',
  mood_anxious:   'mental_and_emotional_health',
  mood_irritable: 'social_stress',
  acne:           'other_pain',       // closest available item (v2: add dedicated acne item)
  headache:       'other_pain',
  back_pain:      'other_pain',
};
```

### useEukiContent Hook

```typescript
// apps/mobile/src/hooks/useEukiContent.ts
export function useEukiContent(phase: CyclePhase | null, dayInCycle: number) {
  // dayInCycle: 0-indexed integer, days elapsed since cycle start date,
  // as returned by usePredictions. Day 0 = first day of period.
  const content = getEukiContent();

  // Phase unknown (new user, <1 completed cycle, mid-first-period, or irregular):
  // fall back to 'unknown' mapping — generic 'menstruation_period' item.
  const resolvedPhase = phase ?? 'unknown';
  const itemIds = phaseMapping[resolvedPhase];
  const rotatedId = itemIds[dayInCycle % itemIds.length];
  const phaseSnippet = content.getItem(rotatedId) ?? content.getSection('menstrual_cycle_101')!.items[0];

  return {
    phaseSnippet,
    getSymptomContent: (symptom: SymptomKey): EukiItem | undefined =>
      content.getItem(symptomMapping[symptom]),
  };
}
```

**Phase confidence requirement:** `PhaseInsightCard` renders in two states:
- **Known phase** (≥1 completed period start + end logged): renders phase-matched content.
- **Unknown phase** (0 completed cycles, OR mid-first-period — start logged, no end yet, OR irregular cycle with no prediction): renders generic `menstruation_period` item with caption "Learning about your cycle." The card never renders a phase label it is not confident about.

In the mid-first-period state: `usePredictions` returns `phase: null` until a period end is logged. `useEukiContent` treats `null` as `'unknown'` and falls back to the generic item. No special case needed.

### Surface 1: PhaseInsightCard (LedgerScreen)

- **Prop contract:** `PhaseInsightCard` receives `phase: CyclePhase | null` and `dayInCycle: number` as props from `LedgerScreen`. `LedgerScreen` calls `usePredictions()` and passes the result down. `PhaseInsightCard` calls `useEukiContent(phase, dayInCycle)` internally.
- **Position:** between `HorizontalCalendar` and the log list (existing scroll area, no layout restructure)
- **Visual:** phase color from `docs/design.md` as left-border accent. Phase icon (Material Symbols: `water_drop`, `psychiatry`, `wb_sunny`, `mode_night`). Phase label. Snippet from `phaseSnippet.content` truncated at 180 chars. "Read more →" link.
- **Rotation:** `dayInCycle % itemIds.length` — different snippet on each day within the phase window.
- **Tap / "Read more →":** opens `ContentSheet` (LedgerScreen is full-screen, no stacked modal conflict).
- **Unknown phase / new user:** renders with neutral color + `menstruation_period` snippet. Caption: "Learning about your cycle."

### Surface 2: SymptomChips + Inline Link (DataEntryModal)

**New layout for DataEntryModal:**

```
Row 1: [ Start |→ ]  [ →| End ]
Row 2: Bleeding: [ Spotting ] [ Light ] [ Medium ] [ Heavy ]
       (segmented control, single-select, Locket Blue when active)
Row 3: Clots: [ Small clots ] [ Large clots ]
       (appears only when bleeding intensity is selected, animates in — fade/expand)
Row 4: Symptoms (multi-select pills, Locket Blue bg when active):
       [ Cramps ] [ Bloating ] [ Nausea/Fatigue ]
       [ Mood ↓ ] [ Mood anxious ] [ Irritable ]
Row 5: [ Add Notes... ] (existing text input)
Row 6: [ Save Note ] (existing)
```

After selecting a symptom chip: inline caption appears below the chip row:
`Cramps  →  Why does this happen?` (tappable, Locket Blue, caption size)

Tap: opens `ContentSheet` with the matching Euki item.

**Navigation note:** `ContentSheet` opens as a nested React Native `Modal` (RN core, not React Navigation) over `DataEntryModal`. RN nested `Modal` is supported on RN ≥ 0.71 on iOS and Android. `DataEntryModal` uses `<Modal animationType="fade" transparent>`, `ContentSheet` uses `<Modal animationType="slide">` nested inside it. No React Navigation stack required.

### ContentSheet (Shared Component)

- Bottom sheet / full-screen modal
- Header: Euki item title
- Body: full `content` text, scrollable
- Links section (if present): external resource links via `Linking.openURL()`. On failure: show inline toast "Couldn't open link." Do not crash.
- Footer: `Educational content from Euki (euki.app)` — tappable, opens euki.app in browser
- Dismiss: swipe down or ✕ button

### Medical Disclaimer (One-Time Acknowledgment)

First time any `ContentSheet` opens, a `DisclaimerModal` appears first:

```
"Locket's educational content is for informational purposes only and is
not a substitute for professional medical advice, diagnosis, or treatment.
Always consult a qualified health provider with questions about a medical condition."

[ Got it ]
```

Persisted in `AsyncStorage` as `content_disclaimer_seen: true`. The flag is written **only on "Got it" tap** — not on modal display. Force-quitting during the disclaimer shows it again on next content tap. This is correct behavior.

## Open Questions

1. **Snippet length in PhaseInsightCard:** 180-char truncation with "Read more →" is the current assumption. Full paragraph is an alternative if the card feels too terse.
2. **Symptom-weighted phase card:** Should the phase card prioritize content matching the user's most recently logged symptom? Phase-based rotation is simpler for MVP. Symptom-weighted is a v2 idea.
3. **Bleeding product chips:** Defer the 7 Euki product types to v2 to keep the DataEntryModal uncluttered. A "Products used" sub-row is the suggested v2 pattern.
